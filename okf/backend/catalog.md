Catalog is the identity-and-metadata bounded context for a user's saved music. Its two aggregate roots are `Track` (`domain/track.go`) and `Playlist` (`domain/playlist.go`), both identified by wrapped-UUID value objects (`TrackId`, `PlaylistId`) and owned by a `shared.UserId`.

`Track` carries title/artist/album plus optional metadata (year, genre, track_number, album_artist, isrc, duration), an ordered `FeaturedArtists []FeaturedArtist` of guest ("feat.") credits, and an `AcquisitionStatus` enum (`AcquisitionPending`/`Ready`/`Failed`, zero-value = pending). The status/`AudioRef`/`FailureReason` invariant is enforced only through aggregate methods — `MarkReady(audioRef)`, `MarkFailed(reason)`, `RevertToPending()`, `IsStreamable()` — never by direct field mutation. `NewTrack` computes a `DedupKey` (`domain/dedup.go`: lowercase, NFKC-normalized, alphanumeric-stripped `title|artist|album`) used by the persistence adapter's `ON CONFLICT (user_id, dedup_key) DO NOTHING` upsert, so re-saving the same track by metadata is idempotent and returns the existing row (`TrackRepository.Add` returns `(stored, created, err)`).

`FeaturedArtist` (`domain/featured_artist.go`, ADR-0019) is an immutable value object — display name plus optional canonical ids (MusicBrainz MBID and/or Deezer artist id), role always `"featured"` in v1. Its `IdentityKey()` (MBID, else `dz:<deezer_id>`, else `name:<normalized name>`) is the stable grouping key for "everything featuring X" and deliberately mirrors the generated `identity_key` column on the `featured_artists` table (see [tracks-table](../data/tracks-table.md)) so Go-side and SQL-side identity agree. `NewFeaturedArtist` trims and rejects empty names so unnamed credits are dropped at construction.

`Playlist` holds an ordered `[]PlaylistTrack{TrackId, Position}` and enforces contiguous 0..N-1 positions and no-duplicate-track invariants through `AddTrack`/`RemoveTrack`/`Reorder` — never via raw slice manipulation. `ErrTrackAlreadyInPlaylist` and `ValidationError` are the domain's `CodedError`-carrying error types (HTTP status travels with the domain error via `HTTPStatus()`).

Ports (`ports/`): `TrackRepository` (now also `SetTrackNumber` — fill-only, `WHERE track_number IS NULL`, so it never overwrites a real value and is idempotent; `ReplaceFeaturedArtists` — full-set replace used by the backfill; and `ListTracksFeaturing` — identity-key match), `PlaylistRepository` (composed of `PlaylistStore` + `PlaylistTrackMutator`, split for ISP), `AudioStore` (`Exists`/`Store`/`Stream`/`Delete` plus an `AudioStream` = `io.ReadSeeker`+`io.Closer`), `AudioURLSigner` (an *optional* capability an `AudioStore` may implement — `PresignGet(ctx, audioRef, ttl)` mints a short-lived direct-from-storage URL; detected with a type assertion, never required, so the filesystem store simply doesn't presign), `FeaturedArtistResolver` (resolves "feat." credits from the discovery providers; implemented by a bridge adapter, never a direct catalog→discovery import), and `AcquisitionScheduler` (a one-method port the acquisition context implements, injected into catalog services so `AddTrackService` and `StreamTrackService` can trigger re-acquisition without importing acquisition).

Services (`service/`): `AddTrackService` (validates, dedups, copies the request's featured credits onto the track before `Add`, publishes `track_added_to_library`, schedules acquisition for a freshly-created track — the event now embeds the *full* wire-shaped track via `trackAddedPayload` (F10) so a receiving client inserts the row directly instead of refetching; the legacy `track_id` key is retained for older clients, and keys are kept in sync with the handler's `TrackResponse`), `ListTracksService`, `DeleteTrackService` (deletes the row then best-effort deletes the audio file, logging orphans rather than failing), `SetTrackNumberService` (persist-as-you-browse: writes back an album position the client derived from the album tracklist, fill-only at the repo so re-running never clobbers), `AudioURLService` (`audio_url.go`: mints presigned URLs — 1-hour TTL — per streamable track so the native player streams directly from storage instead of proxying every byte through the API; a nil signer or non-streamable/unknown track yields silence, and a per-track presign failure skips only that track — the client falls back to the proxy endpoint for anything absent), `BackfillFeaturedService` (`backfill_featured.go`: pages the user's library 200 at a time, resolves credits per track via the resolver port, and `ReplaceFeaturedArtists` — idempotent; a per-track resolver error is logged and skipped so one bad lookup doesn't abort the run), `ListFeaturingService` (the "everything featuring X" browse), `PlaylistService` (create/list/get/rename/delete + track membership; now emits `playlist_renamed`, `track_removed_from_playlist`, and `playlist_reordered` events alongside the existing ones; a documented AIDEV-NOTE flags a narrow non-transactional race in `AddTrack`), and `StreamTrackService` (serves audio, and on a storage-miss reconciles the track to `Failed` and reschedules acquisition via `recoverMissingAudio`; its `RecoverIfMissing` is the client's playback-error hook for *presigned* streams, which bypass `Execute` and its safety net — it re-checks `Exists` and only acts when the object is genuinely absent, so a transient network failure is a no-op with no spurious re-acquisition).

Adapters: `adapters/handler/` (chi handlers, all auth-gated via `auth.RequireUserID`): `TrackHandler` (routes now include `GET /featuring`, `POST /featured-backfill` — synchronous, the library is small — and `PATCH /{trackId}/track-number`; `TrackResponse` carries `featured_artists`), `featured_artist_dto.go` (`FeaturedArtistDTO` wire shape — optional ids are pointer-typed and omitted when unknown so absence stays distinct from a zero id), `AudioURLHandler` (`POST /audio-urls`, batch capped at 200 so one request can't sign thousands of objects; unsignable tracks are simply absent from the response), `PlaylistHandler`, and `StreamHandler` (now sets the Content-Type from the audio ref's extension via `audioContentType` — iOS/expo-audio decodes progressive audio by Content-Type, so an m4a sent as `audio/mpeg` fails to play; also `HandleRecover`, mounted at `POST /tracks/{trackId}/audio/recover`, returning 202). `adapters/discoverybridge/` (new): `FeaturedResolver` satisfies the catalog `FeaturedArtistResolver` port by delegating to the discovery resolver and translating discovery value objects into catalog ones — mirrors `playback/catalogbridge`, keeping catalog free of a discovery import. `adapters/persistence/` (`PgxTrackRepository`, `PgxPlaylistRepository` over pgx, sharing a `trackScanDest()` column-order helper so playlist-track joins reuse the exact track scan; `Add` now runs in a transaction so the track insert and its featured-artist links commit atomically; `featured_artist_repo.go` holds the featured machinery — `writeTrackFeatured` upserts canonical artists on the generated `identity_key` and links them in position order, and `loadFeaturedForTracks` batch-attaches credits in one query, avoiding an N+1, on `GetByID`/`ListForUser`/`GetByDedupKey`/`ListTracksFeaturing`; playlist-track joins do not load credits). `adapters/storage/` (`FilesystemAudioStore` with path-traversal-safe `safePath` and cross-filesystem-safe rename-or-copy, and `ObjectStorageAudioStore` over MinIO/S3, which also satisfies `AudioURLSigner` — presigning is a local HMAC over the request, no network call to storage, so it's cheap enough to mint per track at queue-build time).
