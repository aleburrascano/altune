Playback (server) is deliberately thin: per ADR-0010, the live `Queue` (advance/prev/shuffle/repeat) is owned entirely by the mobile client (`apps/mobile/src/shared/playback/queueStore.ts`, see [shared-playback](../mobile/shared-playback.md)). This backend context persists only a snapshot of it — the server half of the Memento pattern used for resume-on-reopen.

**Domain** (`domain/queue_state.go`): `QueueState{UserId, TrackIds []string, CurrentIdx, PositionMs, Shuffled, RepeatMode, SourceId, NaturalOrder []string, UpdatedAt}`. `TrackIds` is deliberately `[]string`, not `[]catalog/domain.TrackId` — an AIDEV-DECISION documents that playback references catalog tracks by id across the context seam rather than coupling to catalog's identity type for a snapshot the server never reasons about domain-wise. `TrackIds` is the queue in *play* order; `NaturalOrder` is the same ids in their pre-shuffle (album/playlist/library) order, letting the client rebuild the exact shuffled sequence AND un-shuffle back to the original order after relaunch — opaque to the server, carried through and never reasoned over, empty for older rows/clients (migration `009_playback_queue_natural_order.sql`).

`RepeatMode` is a three-state enum (`RepeatOff` zero-value / `RepeatAll` / `RepeatOne`) with `String()`/`ParseRepeatMode`, mirroring the mobile `RepeatMode` union.

All construction funnels through the unexported `newQueueState`, the single invariant gate: `PositionMs >= 0`; an empty `TrackIds` normalizes `CurrentIdx` to 0; a non-empty one requires `CurrentIdx` in range; nil `trackIds`/`naturalOrder` are normalized to `[]string{}` so callers and JSON serialization never see nil. `naturalOrder` is a positional constructor parameter (it is always supplied by both construction sites — a functional option made it look optional and was removed). Gate violations return `*domain.ValidationError`, which structurally implements `httputil.StatusError` (status 400 as a plain int — no `net/http` in the domain); `ParseRepeatMode` rejects unknown modes the same way. Three public constructors funnel through the gate — `NewQueueState` (fresh, stamps `UpdatedAt=now`), `RehydrateQueueState` (reconstitutes from storage, preserving stored `UpdatedAt` — so a corrupt row can't produce an invalid `QueueState`), and `EmptyQueueState(userId)` (the canonical "no queue" value, used whenever no row exists for a user).

**Ports**: `ports/queue_state_repo.go` defines `QueueStateRepository{Upsert, GetForUser}` — a 2-method port, one per operation, no read/write coupling. `ports/now_playing.go` defines `NowPlayingReader{Lookup(ctx, userId, trackId)}` returning `(*NowPlayingTrack, bool, error)` — the bool is false with a nil error when the track is absent (deleted, unknown, or malformed id), which resume treats as "no snapshot", not a failure. `NowPlayingTrack` is a minimal display-ready snapshot (Id/Title/Artist/ArtworkURL/DurationSeconds/AcquisitionStatus), mirroring catalog track data by value across the context seam.

**Service** (`service/queue_service.go`): `QueueService.Save` parses the wire `RepeatMode` string, constructs a `QueueState` (rejecting invalid input before any I/O), and upserts. `QueueService.Resume` returns the saved snapshot or `EmptyQueueState` — callers never receive nil, so the handler needs no nil-check. `ResumeView` wraps Resume plus the currently-playing track's display snapshot (`ResumeView{State, CurrentTrack}`), so the client can render now-playing from this one small call instead of blocking on a full library rehydrate; `CurrentTrack` is nil when the queue is empty, the index is out of range, or the track can't be found. `NewQueueService(repo, nowPlaying)` takes both dependencies as required parameters — a miswired composition root fails at construction instead of silently omitting `CurrentTrack` (tests pass an in-memory fake).

**Adapters**: `adapters/handler/queue_handler.go` exposes `PUT /queue-state` (save — request now includes `natural_order`) and `GET /queue-state` (resume, via `ResumeView`) behind `auth.RequireUserID`, mapping wire DTOs (snake_case JSON) to/from the service input/output; the response carries `natural_order` and an embedded `current_track` (`omitempty` — id/title/artist/artwork_url/duration_seconds/acquisition_status). Errors go through `httputil.HandleServiceError`: domain `ValidationError`s render as 400 with their message; anything else is logged and returned as a generic 500. `adapters/catalogbridge/now_playing_reader.go` implements `NowPlayingReader` against catalog's `TrackRepository` *port* (never catalog's adapters — the standard cross-context integration seam), translating a catalog Track into playback's display snapshot; a malformed track id is treated as absent, not an error, so resume degrades to "no snapshot" instead of failing the whole call. `adapters/persistence/queue_state_repo.go` (`PgxQueueStateRepository`) is a single-row-per-user upsert (`ON CONFLICT (user_id) DO UPDATE`) against `playback_queue_state` (see [playback-queue-state-table](../data/playback-queue-state-table.md)), storing `track_ids`/`natural_order` as native array columns and `repeat_mode` as its string form, rehydrating via `domain.RehydrateQueueState` (with `naturalOrder` positional) so the same invariant gate applies on read.
