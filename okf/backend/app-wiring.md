`internal/app` is the composition root: the only place adapters are chosen and injected into ports, per the project rule that the object graph is wired explicitly. `App` (`app.go`) owns config, the pgx pool, the (nilable) Redis client, the HTTP server, and every lifecycle-bound background component (acquisition scheduler, vocab refresh, event bus/feed, alert monitor, log ring, provider health, eval meter). `setup(ctx)` assembles in order: pools тЖТ auth verifier тЖТ event bus тЖТ catalog+acquisition тЖТ playback тЖТ discovery content/consensus тЖТ search service тЖТ background tickers тЖТ enrichment тЖТ discovery handler тЖТ vocab refresh тЖТ chi router тЖТ admin (Mission Control) тЖТ alert monitor. Event publishers (catalog/acquisition services) receive the bus wrapped in admin's `eventtap.Tap` decorator тАФ the Mission Control system-wide tap тАФ while the per-user SSE handler subscribes to the inner bus directly; the database is *not* nil-tolerant: `database.NewPool` errors on an empty `DATABASE_URL`, so a misconfigured process fails at startup instead of panicking per request. **Nil-tolerant degradation pervades**: nil Redis, nil MusicBrainz, nil audio store, nil enrichment all switch features off rather than fail тАФ e.g. without an audio store the acquisition scheduler is nil, the retry endpoint isn't mounted, and shutdown falls back to bare `wg.Wait()`. Playback's `NowPlayingReader` is deliberately *not* in that set: it is built from the always-present catalog `trackRepo` and passed to `NewQueueService` as a required parameter, so a miswired graph fails at construction instead of silently dropping `current_track` from every resume.

**Search wiring** (`search_wiring.go`): `BuildSearchService` is deliberately the *single construction site* for the search pipeline so the offline eval exercises exactly the ranking users see тАФ the eval CLI, the eval meter, and production all call it. `BuildSearchServiceWithTransport(..., rankingOnly)` adds two seams: an injectable transport (record/replay for fixtures) and `rankingOnly`, which skips display enrichment (artwork chain, related-groups тАФ the bulk of a search's HTTP) and the Redis result cache while keeping every rank-affecting flag live; exploration is *never* wired in rankingOnly so eval order stays deterministic. Eval/synthetic searches always get a nil event store so they never pollute telemetry. Ranking experiment flags: `CROSS_KIND_PROMINENCE_ENABLED` defaults **true** (the only one on by default); `TAIL_DEMOTION`, `EXPLORATION`, `BEHAVIORAL_RANKING`, `EVAL_METER` default false. The durable identity store layers Redis as a read-through front over Postgres (source of truth, survives Redis flushes); `RedisEnrichmentCache` does triple duty as enrichment cache + identity bridge + MBID index, so a detail-open warms the search path. There are intentionally **two** vocabulary stores (setup's for suggest/refresh; the search service builds its own) and **two** MusicBrainz adapter instances тАФ rate safety holds because pacing lives in the shared transport, not the adapter.

**HTTP transport** (`http_client.go`, `live_transport.go`): `defaultLiveTransport` is a package-level `liveTransport` shared process-wide тАФ per-host `rate.Limiter`s (musicbrainz 1/s, itunes 0.5/s, last.fm 5/s, ytmusic 2/s; burst 4 so one query's per-kind fan-out isn't limited against itself) plus retry (3 attempts, 429/5xx, linear backoff). It must stay process-shared: per-client limiters would let N providers each do N├Ч a host's limit тАФ exceeding published limits is what got the prod IP throttled (surfaced as spurious search timeouts). `clientFactory`'s zero value uses the live transport; eval injects record/replay, and replay never touches the live transport. YouTube Music is transport-injected (its library owns a package-global client).

**SSE** (`sse_handler.go`): lives in the composition root deliberately тАФ it is the seam between `shared/events` and the auth context, so `shared/events` never imports auth. Hard-won contracts: never reply 204 to an empty `Last-Event-ID` replay (204 tells EventSource to stop reconnecting тАФ a past hot-loop bug), emit `event: resync` when the ring has evicted events in the requested gap, flush `:ok` immediately, heartbeat every 25s.

**Operator surfaces**: three isolated search stacks exist so failures can't cross-contaminate circuit breakers тАФ production `searchSvc` (breaker inside `NewService`), the eval meter's dedicated `BuildSearchService` instance (`eval_runner.go`, off by default per its AIDEV-NOTE; smoke-checks 5 fixed queries against a 0.80 baseline), and Mission Control's re-run (`rerun.go`, builds providers directly with no Service тАФ bypasses breakers by construction, records per-stage waterfalls via an `ExchangeRecorder`). `search_inspector.go` adapts the *live* service for Mission Control test-search (real pipeline, result cache bypassed, no telemetry). Admin routing is two-layer: shell/config public, all data routes behind auth + `OperatorOnly` (fails closed when `OperatorUserID` is unset). The public `/health` exposes ok/degraded only тАФ dependency topology is operator-gated, and "not_configured" is deliberately distinct from "down".

**Lifecycle** (`Run`): SIGINT/SIGTERM via `signal.NotifyContext`; shutdown order with individual timeouts тАФ server (30s) тЖТ alert monitor тЖТ event feed тЖТ eval meter тЖТ vocab refresh (5тАУ10s each) тЖТ acquisition scheduler (30s, or bare `wg.Wait()`) тЖТ pool/Redis close last. Background tickers (behavioral refresh 30min, nightly corpus materialize when `BEHAVIORAL_CORPUS_PATH` set, metrics rollup 6h тАФ always on, alert monitor 30s) bind to the Run context and register on the shared WaitGroup.

**cmd/ reuse**: `cmd/api` runs `app.New(...).Run` (subcommands like `migrate-dedup`/`backfill-duration` deliberately bypass app with their own CLI audio-store builder). `cmd/discoveryeval` reuses `BuildSearchService`/`BuildSearchServiceWithTransport`/`BuildVocabularyStore`/`BuildConsensusProviders` тАФ see [eval-harness](discovery/eval-harness.md). `cmd/discoverytrace` does **not** reuse the builders тАФ its provider set mirrors `buildDiscoveryProviders` by hand (a known drift risk) and skips identity-stamping and artwork.
