`AudioStore` (`ports/audio_store.go`) is the port for reading/writing a track's audio file: `Exists`/`Store`/`Stream`/`Delete`, where `Stream` returns an `AudioStream` (`io.ReadSeeker` + `io.Closer` — the caller must close it). `AudioContentType(audioRef string) string` is the single audio-ref→MIME mapping (`.m4a`→`audio/mp4`, `.opus`→`audio/opus`, `.ogg`→`audio/ogg`, default `audio/mpeg` for legacy mp3 refs), shared by the upload side (object storage sets it on `PutObject`) and the serve side (`StreamHandler` sets the response header) — the two must agree because iOS/expo-audio decodes progressive audio by Content-Type, not by sniffing.

`AudioURLSigner` is an *optional* capability an `AudioStore` may additionally implement: `PresignGet(ctx, audioRef, ttl) (string, error)` mints a short-lived URL the native player streams directly from storage, bypassing the API proxy (which otherwise pays auth + Postgres + two storage round-trips per range request). Detected with a type assertion (`signer, _ := store.(ports.AudioURLSigner)`), never required — the filesystem store simply doesn't implement it, and callers fall back to the proxy stream endpoint.

Two adapters (`adapters/storage/`):
- **`FilesystemAudioStore`** (`filesystem.go`) — local disk under `baseDir`. `safePath` rejects any `audioRef` that isn't `filepath.IsLocal` (blocks path traversal). `Store` prefers `os.Rename`; on `syscall.EXDEV` (source temp dir and destination audio volume on different filesystems — the norm on a Linux VM with a separate mounted volume) it falls back to a copy-then-remove. Does **not** implement `AudioURLSigner`.
- **`ObjectStorageAudioStore`** (`object_storage.go`) — MinIO/S3-compatible client. Implements both `AudioStore` and `AudioURLSigner`. `PresignGet` is a local HMAC-over-the-request operation (no network call to storage), so it's cheap enough to mint per track at queue-build time. `Store` sets `ContentType` via `ports.AudioContentType` on `PutObject`.

Services:
- **`AudioURLService`** (`audio_url.go`) mints presigned URLs (`audioURLTTL` = 1 hour) for a batch of tracks so the native player streams directly from storage. Constructed from an `AudioStore`; if the store doesn't satisfy `AudioURLSigner`, `signer` is nil and `Resolve` returns `(nil, nil)` immediately — every track falls back to the proxy. Otherwise: one `ListByIDs` batch read for the whole request (a narrow `trackBatchReader` interface) instead of a `GetByID` per track — per-id lookups here cost two queries per track on the hottest path after search (every queue build). A track that's unknown, not owned, or not `IsStreamable()` is silently skipped; a per-track presign failure (logged via `slog.WarnContext`) skips only that track, never the whole batch.
- **`StreamTrackService`** (`stream_track.go`) is the proxy path: `Execute` loads the track, checks `IsStreamable()` (else `ErrAudioNotAvailable`), and streams via `AudioStore.Stream`. On a storage miss it calls `recoverMissingAudio`, which re-checks `Exists` and — if genuinely gone — marks the track `Failed` (`track.MarkFailed`, persisted via `Update`) and unconditionally reschedules acquisition (fire-and-forget; scheduling runs whether or not the mark-failed persist succeeded). `RecoverIfMissing` is the separate entry point for *presigned* streams, which bypass `Execute` entirely (they hit storage directly) and so miss its safety net: the client calls it on a playback error, and it only acts when the object is genuinely absent — a transient network failure over a file that's actually present is a no-op, avoiding spurious re-acquisition. Both paths funnel into the shared `reconcileMissingAudio`, parameterized by an already-known `exists`/`err` so `RecoverIfMissing` (which already ran its own `Exists` check) doesn't pay for a second storage round-trip.

Handlers:
- **`AudioURLHandler`** (`adapters/handler/audio_url_handler.go`) — `POST /audio-urls`, batch capped at `maxAudioURLBatch` (200) so one request can't sign thousands of objects; unsignable tracks are simply absent from the response.
- **`StreamHandler`** (`adapters/handler/stream_handler.go`) — `HandleStreamAudio` (the proxy endpoint) sets `Content-Type` via `ports.AudioContentType(*out.Track.AudioRef)` and serves via `http.ServeContent` (range-request support). `HandleRecover`, mounted at `POST /tracks/{trackId}/audio/recover`, calls `StreamTrackService.RecoverIfMissing` and returns 202 — this is the client's playback-error hook described above.
