Two tables in `001_baseline.sql` back the `Playlist` aggregate (see [catalog/playlist](../backend/catalog/playlist.md)). `playlists`: `id UUID PK`, `user_id UUID NOT NULL`, `name TEXT NOT NULL`, `created_at`/`updated_at TIMESTAMPTZ NOT NULL DEFAULT now()`. `playlist_tracks` is the membership join table: `playlist_id UUID NOT NULL REFERENCES playlists(id) ON DELETE CASCADE`, `track_id UUID NOT NULL REFERENCES tracks(id) ON DELETE CASCADE` (see [tracks-table](tracks-table.md)), `position INTEGER NOT NULL`, with composite `PRIMARY KEY (playlist_id, track_id)` (a track cannot appear twice).

The Go aggregate (`playlist.go`) models membership as `PlaylistTrack{TrackId, Position}` value objects held in `Playlist.Tracks []PlaylistTrack`. Read-side projections are no longer fields on `Playlist` itself: `PlaylistSummary{TrackCount, PreviewArtworkURLs}` is a separate struct, and `PlaylistWithSummary{Playlist, Summary}` pairs the two — kept apart so the write model and its behaviour methods stay free of query artefacts. `PreviewArtworkLimit = 4` is the single definition of the preview-tile cap, read by both the SQL projection (`ListForUser`) and the Go fallback `PreviewArtworkURLs(tracks []*Track) []string` (used on the `GetWithTracks` path where tracks are already loaded in Go, selecting up to `PreviewArtworkLimit` distinct artwork URLs in position order). `NewPlaylist` validates the name is non-empty and ≤100 chars (`validatePlaylistName`). Aggregate methods enforce invariants at every mutation: `AddTrack` rejects duplicates via `ErrTrackAlreadyInPlaylist` (a `CodedError` carrying HTTP 409); `RemoveTrack` and `Reorder` renormalize `Position` to a contiguous `0..N-1` range in-memory.

`PgxPlaylistRepository` (`playlist_repo.go`) implements `ports.PlaylistRepository` — one flat interface (the former `PlaylistStore`/`PlaylistTrackMutator` ISP split was collapsed: no consumer ever used a narrow half). `ListForUser` now returns `[]domain.PlaylistWithSummary` and computes both projections in its single query: `track_count` via the shared `playlistTrackCountSubquery` constant, and `preview_artwork` via `COALESCE(array_agg(...), '{}')` over a subquery selecting up to `PreviewArtworkLimit` distinct non-null `artwork_url`s ordered by earliest position — one round-trip for the whole playlists screen, replacing the removed per-playlist `GetPreviewArtwork` method (which was also the port's only tenant-unscoped query). `GetByID` now returns `(*domain.Playlist, domain.PlaylistSummary, error)`, reusing the same `playlistTrackCountSubquery` so single-playlist responses (e.g. rename) report the real count without loading track rows; `GetWithTracks` discards the summary and just needs the playlist. `RemoveTrack` runs in a transaction, delegating the position collapse to the package-level `renumberPlaylistPositions(ctx, tx, playlistId)` helper — shared with `track_repo.go`'s `Delete`, which calls it once per playlist a deleted track belonged to, so the renumbering rule (`UPDATE ... SET position = sub.new_pos FROM (SELECT track_id, ROW_NUMBER() OVER (ORDER BY position) - 1 ...)`) has one definition instead of drifting between the two write paths. `ReorderTracks` batches per-track position updates via `pgx.Batch` inside a transaction.
