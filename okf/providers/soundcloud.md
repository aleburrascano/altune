SoundCloud is altune's sole source for the unreleased/leaked/underground long tail — no mainstream catalog (Deezer, iTunes, MusicBrainz) indexes it. `SoundCloudAPIAdapter` (`soundcloud.go`) talks directly to SoundCloud's internal `api-v2.soundcloud.com` JSON backend (the same one the website calls) rather than scraping HTML or shelling to yt-dlp for search.

**Auth model.** No user auth — gated only by a public `client_id` query param embedded in the site's JS bundles. Client-id resolution now lives in its own file, `soundcloud_auth.go` (moved out of `soundcloud.go`, same logic and names). `clientIDResolver` (`newClientIDResolver`, `soundcloud_auth.go`) fetches `https://soundcloud.com/`, regexes the `/assets/*.js` bundle URLs (`scAssetURLRe`), scans them from the end for `client_id:"<32 alnum>"` (`scClientIDRe`), and caches the result. Concurrent cold-start callers collapse via `singleflight`. On a 401/403 (`isAuthStatus`, still in `soundcloud.go`), the adapter calls `resolver.invalidate()` and re-resolves once. This is reverse-engineered and against SoundCloud's ToS — accepted per `AIDEV-DECISION` in the file for self-hosted personal/family use.

**Capabilities (`soundcloud.go`).** `Search` dispatches per requested kind: `searchTracks` (paginated via `next_href` up to `scMaxResults=40`), `searchAlbums` (`/search/albums`, typed playlists with `set_type` album/ep/single), `searchArtists` (`/search/users`). `searchAlbums`, `searchArtists`, `GetArtistTopTracks`, `GetArtistAlbums`, and `GetRelatedTracks` all now share one generic helper, `scFetchList[T any]`: it runs the auth-retry fetch, decodes one JSON page into `struct{ Collection []T }`, and maps each item via the caller's `mapFn` — only the URL, the item type, and the mapper differ per endpoint (the per-endpoint `scAlbumSearchResponse`/`scUserSearchResponse` page-shape types were dropped, since `scFetchList`'s inline anonymous struct replaces them). `Resolve` implements `ports.ArtworkResolver`, wired last in the artwork chain so it only fires when ID-based sources miss (see [artwork-chain](artwork-chain.md)). `GetArtistTopTracks`/`GetArtistAlbums` implement `ports.ArtistContentProvider` keyed by the numeric SoundCloud user id. `ResolveArtistID` implements `ports.ArtistIDResolver` (name → SC user id via the top `searchArtists` hit): because MusicBrainz's url-relations never assert a SoundCloud id, the identity-first content fan-out ([merge-dedup](../backend/discovery/merge-dedup.md)) would otherwise never reach SoundCloud for an artist opened from another provider — this lets it resolve SC's own id from the artist name once, then query by that id, so an SC-exclusive artist's tracks/albums still join the discography. `GetRelatedTracks` implements `ports.RelatedTracksProvider` via `/tracks/{id}/related`. `ResolvePermalink` turns a pasted SoundCloud URL into a track via `/resolve?url=`, feeding the import/acquisition workflow (see [pipeline](../backend/acquisition/pipeline.md)). `mapSoundCloudAPITrack` lifts a `publisher_metadata.isrc` into the typed `SearchResult.ISRC` field when present, letting an officially-distributed SoundCloud upload merge with the same recording from Deezer/MusicBrainz; it now also lifts `publisher_metadata.album_title` into `SearchResult.Album` and converts the api-v2 millisecond `duration` into `SearchResult.Duration` (seconds), fields previously left unset on SoundCloud results. `getJSON` now issues its request through the shared `getBytes`/`withHeader` helpers in `providerhttp.go` instead of building the `http.Request` inline — same behavior, shared plumbing with the other providers.

**Rate limits / caveats.** `api-v2` throttles; the codebase discipline is "don't run concurrent evals against it." `client_id` rotates a few times/month — self-healing, not per-request. Public-only reach (private/unlisted tracks never surface). `SoundCloudAdapter` (`soundcloud_ytdlp.go`) is the original yt-dlp-subprocess adapter, track-only, retained as `searchFallback` when `client_id` resolution is down (wired as `NewSoundCloudAdapter()`, passed as the `fallback` argument to `NewSoundCloudAPIAdapter`).
